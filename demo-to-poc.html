// phase 1

Pre-Demo Preparation

Before any demo, gather the following from the prospect — ideally confirmed during the discovery phase. Having this in hand lets you frame the deployment conversation naturally rather than starting from scratch afterward.

1

Gather environment details

Before the demo
  • Employee count

    Drives sizing and license scope.

  • Existing identity data sources

    Active Directory, Azure / Entra ID, Okta, AWS, and similar — determines which connectors to configure.

  • Deployment preference

    SaaS or on-premises. Sets the infrastructure and setup path.

// phase 2

Post-Demo Handoff

Same call, or within 24 hours. Momentum is the goal — lock the next step down before it cools.

2

Four moves to close the loop

Same call · within 24 hours
  • Book the deployment date on the call

    Do not leave the next step open-ended. Get a date on the calendar before hanging up, scheduled for morning US time so Nicolas from Saporo's Switzerland-based team can join and assist. It is far easier to move a scheduled call than to chase a commitment later.

  • Duplicate the template folder in the Sales SharePoint

    Navigate to Sales SharePoint > Documents > Customers. Duplicate the template folder and rename the copy to the prospect's name.

  • Confirm POC prerequisites

    Using the POC prerequisites deck from the template folder, confirm or fill in the prerequisites below with the prospect. Adapt the deck to their specific environment before sending.

  • Grant documentation access

    In HubSpot, open the prospect's contact profile and enable the "Public Documentation" attribute. This gives them access to Saporo's documentation and resources to download the image needed to install. Send the link directly in your follow-up email — include a PDF export or direct link. Do not assume they will find it on their own.

// phase 3

The Deployment Call — Day 1

A guided, hands-on walkthrough conducted on the prospect's machine. Schedule for morning US time so Nicolas from Saporo can join from Switzerland.

3

Deploy and kick off the first run

Day 1 · 1 hour
  • Walk through installation end-to-end

    Conduct the full install live on the prospect's machine during the call.

  • Start the first analysis

    The POC officially begins once the machine is deployed and the first analysis run is initiated.

  • Review initial results if time allows

    The first analysis typically completes in under an hour. If there is time on the same call, begin reviewing initial results.

The clock starts here. The five-business-day window begins at the deployment call, not at the demo — anchor the prospect's expectations to Day 1 being deployment day.

// phase 4

POC Execution Window

The standard POC runs five business days — enough time for the prospect to evaluate meaningfully while keeping the urgency to reach a decision. A focused window produces better outcomes than an open-ended evaluation.

Day 1

Deployment call

Machine live, first analysis running. The POC clock starts.

Days 2–3

Initial results review

Explore the platform, walk through findings, and answer baseline questions.

Days 4–5

Deep dive & proposal

Address additional technical questions, validate against their environment, and move to proposal.

Default to a decision at the end of Day 5. Extensions and additional accommodations are available on a case-by-case basis, but the default expectation should hold.

// phase 5

Post-Deployment Review

Schedule the first review call as soon as the initial analysis completes. The goal is to maintain momentum from Day 1 onward — do not let time pass between deployment and the first look at results.

5

Keep the momentum

Immediately after first analysis

Book the first review while you are still on the deployment call, before leaving. The fastest path to a decision is an unbroken chain of scheduled touchpoints from deployment through proposal.

// compliance & documentation

Handle These Proactively

Address each of these before a prospect raises it. Getting ahead of compliance questions keeps the five-day window on track.

NDAs

Saporo can sign an NDA if the prospect requires it, but the preference is to avoid where possible. For on-premises deployments this is often straightforward to resolve: the platform and all data live entirely within the prospect's own infrastructure, so nothing leaves their environment that would require protection.

Security questionnaires

A dedicated LLM-trained platform is available to help draft answers. Use it to accelerate turnaround rather than answering from scratch.

Certifications

ISO 27001Available now
SOC 2In progress · est. Aug 2026

Documentation access

Enabled in HubSpot on the contact's profile, not via the support portal. Send the link proactively with a PDF export or direct link — customers rarely seek it out on their own.
